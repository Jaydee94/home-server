# Homepage Dashboard Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy gethomepage/homepage als zentrales Dashboard unter `http://home.homeserver` mit Live-Widgets für Kubernetes-Metriken, ArgoCD-Status und Grafana-Dashboards.

**Architecture:** Upstream Helm Chart (`jameswynn/homepage`) als ArgoCD-App unter `argocd/apps/homepage/`. Alle Homepage-Configs (Services, Widgets, Settings) leben in `values.yaml` als GitOps-Quelle. Widget-Credentials kommen aus einem SealedSecret im Namespace `homepage`. ArgoCD readonly-User wird über das bestehende Ansible-Template konfiguriert. dnsmasq-Wildcard `*.homeserver` deckt `home.homeserver` automatisch ab — kein DNS-Update nötig.

**Tech Stack:** Helm (jameswynn/homepage), ArgoCD GitOps, Traefik Ingress, Sealed Secrets (kubeseal), Ansible (ArgoCD Helm values)

## Gotchas

- **Cross-namespace Secrets:** Kubernetes erlaubt kein `envFrom` über Namespace-Grenzen.

- **Icons:** selfh.st/icons — Icon-Dateinamen ohne Pfad angeben (z.B. `grafana.svg`).

  deckt `home.homeserver` automatisch ab.
